# AI 算力系统连接图谱重构设计

## 背景

上一版已经去掉左侧目录，并把内容改为全景海报卡片。但用户继续反馈三个核心问题：看不到模块之间怎么连接、关系模式筛选没有实际意义、公司数据可以后续统一接入。新的重点不再是参考小红书海报的背景风格，而是像系统剖面图一样说明“材料、设备、芯片、封装、互联、服务器、集群”如何连接成 AI 算力系统。

## 目标

把主界面从“节点卡片墙”改成“AI 算力系统连接图谱”。页面应直接展示模块关系，不再要求用户通过筛选按钮理解关系。现有能力继续保留：节点详情、公司详情、行情/PE 占位、供需关系、搜索、URL 分享。

## 信息架构

主画布分为四块：

1. 材料输入层：晶圆/衬底材料、PCB/高频材料、光通信材料、封装材料。材料是进入产品本身的物料输入。
2. AI 算力主链路：AI 芯片/HBM → 先进封装 → 板级高速连接 → 交换 ASIC → 光电器件 → 光引擎/CPO/光模块 → 服务器/交换机 → 集群/应用。
3. 制造使能层：半导体前道设备、先进封装设备、PCB 制造设备、光模块/CPO 设备。设备通常不进入最终产品，但决定制造能力、良率与产能。
4. 关键连接说明：用文本化连线说明物料输入、模块集成和制造能力约束，例如“磷化铟衬底 → 光芯片”“半导体前道设备 ⇢ AI 芯片 / 光芯片”。

## 交互

- 去掉左侧层级目录；
- 去掉顶部关系模式筛选；
- 顶部保留搜索和更新时间；
- 点击真实节点打开已有节点详情抽屉；
- 点击节点详情中的公司打开公司详情抽屉；
- 搜索时只保留匹配节点、匹配公司所属节点、以及当前选中节点；
- 旧 URL 中的 `layer` 和 `mode` 参数继续被解析并保留，保证旧链接兼容，但不再作为主画布过滤条件。

## 视觉设计

- 保留温暖纸张背景，但弱化装饰性海报感；
- 主画布更像系统架构图：分层、横向主链路、连接箭头、说明卡；
- 材料层用实线语义，表示进入下游产品；
- 设备层用虚线语义，表示制造能力/产能/良率约束；
- 节点卡片更紧凑，强调模块名称、简短作用和已建模代表公司；
- 设备和部分应用环节先作为解释性虚拟节点，不做公司/行情入口。

## 响应式

- 桌面：完整宽画布，右侧详情抽屉覆盖局部画布；
- 手机：保持大画布横向拖动，不压缩成不可读小图；详情从底部弹出；
- 搜索与点击节点仍然可用。

## 暂不做

- 不扩充公司池；
- 不伪造公司、股价、市盈率或供应关系；
- 不接实时行情 provider；
- 不改变数据库/API 边界；
- 不把整张图做成单张静态图片。
