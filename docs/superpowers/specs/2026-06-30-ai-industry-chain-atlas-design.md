# AI 全产业链研究图谱网页设计

## 1. 用途与边界

本项目用于个人学习和投资研究复盘，以可在电脑和手机访问的交互网页梳理全球 AI 全产业链。内容同时服务两种阅读方式：初次浏览者用全景图建立技术框架，研究者点开节点查看产业位置、公司映射、供应关系、景气驱动、风险和最新市场数据。

网页不提供买卖建议，不保存券商账户或身份信息，不提供交易功能。页面明确注明产业内容更新时间、行情时间戳、来源口径及“仅供个人学习复盘，不构成投资建议”。

## 2. 交付形式与技术栈

- 项目根目录为 `money/ai_industry_chain_atlas`，自身为独立 Git 仓库。
- 使用 Next.js + TypeScript 构建响应式网页和服务端 API。
- 使用 Supabase 托管 PostgreSQL、迁移脚本和服务端数据访问；本地开发可连接独立开发库。
- 使用定时任务更新基本面与缓存行情，网页打开或用户手动刷新时由服务端获取最新或延迟行情。
- 首版支持本地运行；生产环境可部署到 Vercel，并适配手机浏览器与 PWA 安装。
- AI 生成的示意图压缩为 WebP/AVIF，放入版本化静态资源目录，不依赖运行时生成。
- API 密钥只存于本地 `.env.local` 或部署平台的服务端环境变量，永不提交 Git 或发送给浏览器。

## 3. 内容范围

全图覆盖全球产业链及全球代表公司，重点标注美股、A 股和港股，同时保留台股、韩股、日股、欧洲市场及未上市龙头，目标为 120–140 个研究节点，仍以七层呈现，但每层按可投资和可验证的产业分工继续细分：

1. 原材料与关键耗材（18–22 个）：半导体硅片、光刻胶、掩模版、电子特气、湿电子化学品、CMP 材料、溅射靶材、封装基板、ABF 膜、BT 树脂、低损耗树脂、电子级玻纤布、铜箔、锡球与键合材料、InP/GaAs/硅光材料、光纤预制棒、稀土永磁、冷却液等。
2. 制造设备、代工与封测（20–24 个）：光刻、刻蚀、薄膜沉积、清洗、离子注入、热处理、CMP、涂胶显影、量测检测、掩模设备、晶圆制造、先进制程代工、成熟制程代工、封装设备、测试设备、传统封测、2.5D/3D 先进封装、Chiplet、CoWoS 类产能等。
3. 核心芯片、存储与电源（18–22 个）：GPU/XPU、云厂商定制 ASIC、CPU、DPU/IPU、SmartNIC、交换 ASIC、SerDes/Retimer、HBM、DDR、NAND、企业级 SSD、光 DSP、TIA/Driver、激光器、调制器、硅光芯片、PMIC、VRM 与电源管理等。
4. 高速互联、PCB 与网络（18–20 个）：高端 CCL、高多层 PCB、HDI、封装基板、连接器、高速背板、DAC、AEC、可插拔光模块、LPO、CPO、光引擎、FAU/MPO、单模与多模光纤、InfiniBand、以太网交换机、Scale-up、Scale-out 网络等。
5. 服务器、电力、制冷与数据中心（18–22 个）：AI 服务器、服务器 ODM、整机品牌、机柜与母线、存储系统、液冷冷板、CDU、浸没式液冷、风冷、PSU、BBU、UPS、变压器、开关柜、燃气轮机、备用电源、储能、数据中心运营、云算力租赁与电网接入等。
6. AI 平台与软件基础设施（12–16 个）：公有云、GPU 云、基础模型、开源模型、数据标注与合成数据、向量数据库、MLOps、训练框架、推理引擎、模型服务、AI 安全、可观测性、Agent 平台与开发工具等。
7. AI 应用与端侧设备（18–24 个）：办公、编程、搜索、营销、客服、设计、教育、医疗、金融、法律、工业软件、机器视觉、自动驾驶、智能座舱、机器人、无人机、智能手机、AI PC、可穿戴设备、内容娱乐与垂直 Agent 等。

每个节点采用统一数据结构：

- 名称、英文名、所属层级和一句话定义；
- 技术解释与产业价值；
- 直接上游、直接下游及关系类型；
- 核心壁垒、景气驱动和主要风险；
- 全球代表公司、上市市场、股票代码、业务角色和代表产品；
- 技术示意图或场景图；
- 信息来源或核对口径。

每个材料及关键耗材节点至少配置 2 家代表龙头，成熟赛道目标为 3–5 家；公司必须注明其在该节点的具体产品或能力。代表公司用于说明产业位置，不表示覆盖全部参与者，也不构成推荐。对同时跨越多个节点的公司可重复出现，但需注明其对应业务。

## 4. 信息架构与关系模型

主界面采用“分层研究地图”，而非自由拖拽知识图谱。左侧为七层导航和市场筛选，中间为当前层级的节点关系画布，右侧为选中节点的详情抽屉。

产业节点关系使用有向边表达，至少区分三类：

- `supply`：材料、器件或服务的直接供给；
- `integrate`：部件被集成进模块或系统；
- `deploy`：平台或系统部署到下游场景。

默认仅展示当前层级的主干关系，避免 120–140 个节点同时连接造成视觉混乱。选中节点后，高亮它的一阶上游、一阶下游及连接路径，弱化无关节点。用户可切换“供给依赖”“价值流”“全部关系”，并可回到全局概览。

跨层关系通过画布边缘的入口/出口和面包屑路径表达。例如选中 CPO 时，显示“光学材料 → 光芯片 → CPO → 交换系统 → AI 集群”，同时允许点击路径中的节点跨层跳转。

公司间供应关系与产业节点关系分开建模。每条供应关系包含供应方、客户方、供应产品、对应产业节点、适用平台或项目、关系状态、公布日期、证据来源、最后核对日期和置信等级。关系状态至少包括“公司确认”“客户/供应商确认”“监管或招标披露”“多来源报道”“市场推测”；默认图谱只显示前三类，市场推测必须显著标识且不能伪装为已确认事实。

同一供应关系允许存在有效期和历史版本，避免新合作覆盖旧记录。公司详情页可查看其上游供应商、下游客户、共同平台和证据时间线。

## 5. 视觉设计

整体采用克制的深色研究终端风格：深蓝黑画布、低对比度分隔线、米白色详情抽屉，以及单一琥珀橙作为选中与高亮色。视觉目标是“产业研究图册”，而不是实时行情终端。

- 顶栏：产品标题、更新时间、搜索框和视角切换。
- 左栏：七层导航、市场筛选及硬件/软件筛选。
- 主画布：节点、方向箭头、层级标题、路径说明和关系图例。
- 详情抽屉：节点标题、AI 技术示意图、解释、上下游、壁垒、驱动、风险和代表公司。
- 公司详情：业务节点、代表产品、最新行情、估值指标、已验证供应关系和来源时间线。
- 移动端：左栏收进筛选抽屉；主画布允许横向浏览；节点详情改为底部全屏面板。

不使用大面积渐变、霓虹光效、无意义指标卡、实时价格滚动条或厚重的嵌套卡片。所有按钮、搜索、筛选、关系切换和公司代码均为代码原生文字，不能烘焙进图片。

## 6. AI 图片策略

AI 图片采用统一的“编辑式技术剖面图”风格：深色或浅灰背景、精确机械结构、局部琥珀橙强调、无品牌标识、无伪造参数。每张图均标注“AI 生成技术示意图”，避免被误认为实物照片或厂商官方资料。

约 15 个核心硬件节点制作专属示意图，优先覆盖 GPU、HBM、先进封装、光芯片、光模块、CPO、PCB、交换机、AI 服务器、液冷、供配电、IDC、端侧芯片、自动驾驶计算平台和机器人。软件节点使用工作流图，应用节点使用场景图。非核心节点可复用同一视觉体系的类别插图，但不能用图片替代结构化说明。

图片只负责建立直觉，不承担精确技术结论。需要精确表达的接口、关系、公司和风险全部使用网页文字与 SVG 关系线呈现。

## 7. 核心交互

1. 搜索节点、技术名词、公司名称或股票代码。
2. 按产业层级、市场、硬件/软件筛选。
3. 点击节点后打开详情抽屉并高亮一阶上下游。
4. 点击上游、下游或路径节点可跨层跳转。
5. 切换关系类型，查看供给依赖、价值流或全部关系。
6. 在详情抽屉中切换“技术解释”和“投资视角”。
7. 点击“仅看相关公司”后，在当前节点范围内筛选公司标签。
8. 支持键盘焦点、Escape 关闭详情以及减少动画偏好。
9. 点击公司可查看股价、涨跌幅、市值、PE、数据时间戳和供应关系网络。
10. 支持手动刷新行情；失败时显示最近缓存及其时间，不用旧数据冒充实时数据。

筛选与选择状态保存在 URL 查询参数中，便于手机分享和恢复视图；不保存个人身份数据。

## 8. 公司、行情与估值数据

公司主数据与行情快照分离。公司主数据包含法定名称、常用名、ISIN（如可用）、交易所、原始股票代码、统一代码、币种、上市状态和业务节点；行情快照包含价格、涨跌幅、成交时间、数据延迟、币种和来源；基本面快照包含市值、EPS、TTM PE、Forward PE（数据源支持时）、财报期和更新时间。

- 网页打开时通过服务端代理刷新最新或延迟行情，并标注延迟分钟数和市场状态。
- 每日定时更新公司基本面、TTM PE、市值和最近交易日收盘数据。
- PE 使用 `股价 / TTM EPS` 口径；TTM EPS 小于等于零时显示 `N/A`，不显示容易误导的负 PE。
- 所有数值保留来源、抓取时间、交易时间、币种和原始值，展示层再做格式化。
- 数据供应商通过适配器接口接入；首版只实现一个主供应商和缓存回退，后续可替换而不改页面组件。
- 免费或低成本数据用于开发验证；上线前根据美股、A 股、港股覆盖、延迟、授权和预算选择正式套餐。

## 9. 数据与来源口径

节点定义和公司映射优先使用公司官网、年度报告、投资者材料、技术文档、行业标准组织和监管披露。对快速变化的技术进度只描述已公开事实，并记录内容核对日期。公司“代表性”基于其公开业务位置，不根据短期股价表现排序。

网页底部提供来源说明和主要参考链接。正文不大段复制来源内容；技术解释使用转述。对于尚未规模量产的路线，应使用“研发”“验证”“预计”或“公开规划”等明确措辞。

供应关系优先使用公司公告、年度报告、投资者材料、监管披露、招标文件和双方新闻稿。每条关系都必须保存可访问来源；无法公开验证的传闻不进入默认关系图。

## 10. 系统架构与数据边界

系统采用可独立替换的四层结构：

- 展示层：Next.js、TypeScript、响应式组件、PWA 清单和关系 SVG。
- API 层：只读产业查询、公司详情、供应关系和行情刷新接口；服务端校验参数并限制调用频率。
- 数据层：Supabase/PostgreSQL，使用迁移管理 `industry_nodes`、`companies`、`company_node_roles`、`industry_edges`、`supply_relations`、`relation_evidence`、`market_snapshots`、`fundamental_snapshots` 和 `sources`。
- 更新层：定时任务更新基本面和缓存收盘行情；按需接口获取最新或延迟报价；失败时保留最近成功快照并记录错误。

关系画布使用 SVG，以便实现箭头、路径高亮、缩放适配和可访问文本。AI 图片作为静态媒体加载，不与关系图坐标耦合。数据库启用行级安全策略，浏览器只能读取公开研究数据；服务端密钥只在定时任务和 API 路由中使用。

生产部署优先使用 Vercel 托管 Next.js，Supabase 托管数据库。域名和登录不属于首版必需项，但架构不得阻碍后续增加只读公开访问、个人收藏和研究笔记。

## 11. 异常与降级

- 图片加载失败时显示带节点名称的图解占位块，不影响文字内容。
- 搜索无结果时展示清晰的空状态和重置入口。
- 当前筛选隐藏已选节点时，自动关闭详情并提示筛选条件。
- 小屏幕无法完整展示关系图时允许横向滚动，不缩小到不可读。
- 行情供应商超时、限流或停服时返回最近缓存，清楚显示“数据截至”时间。
- 某市场不支持实时行情时自动降级为延迟或最近收盘价，并显示数据等级。
- 定时任务失败时记录运行日志和失败公司，不用空值覆盖最后成功数据。
- JavaScript 被禁用时保留七层节点清单、核心解释和免责声明。

## 12. 验收标准

- 桌面端首屏能识别七层产业结构、当前层、节点关系和详情入口。
- 120–140 个节点均有定义、产业位置、上下游、壁垒、驱动、风险及代表公司字段。
- 每个材料与关键耗材节点至少有 2 家代表龙头，并注明具体业务角色或产品。
- 选中任一节点后，一阶上下游高亮正确，跨层路径可导航。
- 公司供应关系可按证据等级过滤，默认不展示市场推测。
- 至少 15 个核心节点使用风格统一且明确标注的 AI 技术示意图。
- 搜索与各筛选组合工作正常，无结果和重置状态完整。
- 各市场代码采用统一的“原始代码 + 交易所”格式，公司业务映射与公开资料相符；美股、A 股和港股可单独筛选。
- 桌面、平板和手机不存在关键内容裁切或不可恢复的横向溢出。
- 本地开发可运行；生产构建可部署；手机端可访问并恢复分享的筛选状态。
- 打开公司详情时展示最新可用行情、TTM PE 或 `N/A`、数据延迟和时间戳。
- 每日基本面任务可重复执行且不会产生重复快照；单个公司失败不阻断整批任务。
- 页面包含更新时间、来源说明和个人学习用途免责声明。

## 13. 验证方法

1. 运行 Next.js 本地开发服务器并在应用内浏览器打开网页。
2. 检查桌面宽屏、常规笔记本和手机视口。
3. 验证搜索、筛选、节点选择、关系切换、跨层跳转、公司详情、分享 URL 和详情关闭。
4. 对节点数、材料龙头覆盖、关系边、供应关系证据、公司市场代码和必填字段做脚本校验。
5. 使用模拟供应商响应验证最新行情、延迟行情、负 EPS、限流、超时和缓存回退。
6. 验证数据库迁移、行级安全策略、每日任务幂等性和密钥不会进入客户端包。
7. 对 AI 概念设计与最终实现截图进行并排视觉检查。
8. 执行目录结构检查：`find money -maxdepth 2 -print | sort`。
