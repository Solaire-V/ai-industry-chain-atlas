# AI 产业链图谱数据与展示修复设计

## 目标

本次只修复当前主界面与节点库暴露出的数据展示问题，不重做主流程图，不新增公司库大数据，不接实时行情。

修复目标：

1. 消除同一阶段内的重复显示名，避免用户误判为重复节点。
2. 让左侧目录状态进入 URL，支持刷新、分享和直接访问子界面。
3. 对跨阶段复用的真实节点加明确标识，避免误认为数据重复。
4. 对概念节点、待补投资节点、待接行情/供需数据加完整度提示。

## 范围

包含：

- 设备阶段两个“高速测试”节点的显示名拆分。
- `AtlasQueryState` 增加 `view` 字段，并接入左侧目录。
- 节点库展示跨阶段复用标识和出现位置。
- 节点库阶段统计增加待补状态文案。
- 公司库、行情数据、供需关系页面增加真实数据覆盖状态。
- 补充单元测试和组件测试。

不包含：

- 新增公司、行情、供应关系数据。
- 重画主界面 SVG 布局。
- 接入外部行情 API。
- 变更数据库 schema 或 Supabase migration。

## 数据设计

新增一个前端查询状态字段：

```ts
view: "canvas" | "nodes" | "companies" | "markets" | "supply" | "settings"
```

解析规则：

- 未传 `view` 时默认 `canvas`。
- 非法 `view` 归一到 `canvas`。
- 序列化时始终保留 `view`，保持 URL 状态完整且可分享。

跨阶段复用不新增 schema 字段，直接从 `atlasStages` 的 `realNodeId` 引用反推：

- 同一个 `realNodeId` 被多个 stage 引用时，视为跨阶段复用。
- 在节点库卡片和详情页展示它出现在哪些主链模块中。

## 界面设计

### 左侧目录

目录点击后更新 URL：

- 主界面：`?view=canvas&stage=optical-interconnect`
- 节点库：`?view=nodes&stage=equipment`
- 公司库：`?view=companies`
- 行情数据：`?view=markets`
- 供需关系：`?view=supply`
- 数据设置：`?view=settings`

### 节点库

节点卡片：

- 可投资节点：保留“可投资节点”。
- 概念节点：保留“概念节点”。
- 跨阶段复用节点：额外显示“跨阶段”标识。

阶段统计：

- 显示二级分类、细分节点、已绑定投资节点、待补节点。
- 当已绑定投资节点为 0 时，显示“待补投资节点”，避免只显示 `0 个可投资`。

详情栏：

- 增加“数据完整度”或“复用位置”区域。
- 可投资节点继续显示公司与节点详情入口。
- 概念节点继续显示“后续可升级为可投资节点”的状态。

### 数据子页面

公司库：

- 显示总公司数、已挂载节点角色数。

行情数据：

- 明确显示当前行情快照为 0，状态为“行情源未接入”。

供需关系：

- 区分产业链边与公司级供需关系。
- 当前公司级供需关系只有 1 条，页面需要提示“供应关系数据待补全”。

## 正确性要求

- 同一阶段内不允许重复显示名。
- 主界面与节点库必须继续共用同一份 `stageId / realNodeId` 数据。
- URL 中的 `view`、`stage`、`node`、`company`、`q` 能稳定 roundtrip。
- 打开 `?view=nodes&stage=equipment` 应直接进入设备节点库。

## 验证

执行：

- `npm run typecheck`
- `npm run test`

浏览器验收：

- 打开主界面，确认默认仍是画布。
- 打开 `?view=nodes&stage=equipment`，确认进入设备节点库。
- 设备页不再出现两个完全相同的“高速测试”。
- `低损耗 CCL` 或 `AI 集群` 显示跨阶段复用提示。
- 行情数据页明确显示未接入状态。
