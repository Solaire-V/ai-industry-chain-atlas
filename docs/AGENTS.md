# Documentation Rules

## 目录定位

`docs/` 用于保存项目目标、路线图、产品设计决策和实施计划，不放运行时代码、行情数据、密钥或构建产物。

## 子目录约定

- `roadmap/`：保存产品目标、阶段路线、关键取舍和验收标准。
- 后续如需新增 `architecture/`、`research/`、`operations/` 等目录，先在本文件补充用途说明，再新增对应文件。

## 命名约定

- 文档文件名优先使用 `YYYY-MM-DD_topic.md`。
- topic 使用英文小写和下划线，便于搜索和跨平台处理。
- 同一主题的更新优先修改原文件；阶段性变化较大时再新增带日期的新文件。

## 内容约定

- 结论先行，明确目标、范围、非目标、验收标准。
- 面向后续执行者和项目维护者，不记录临时聊天内容。
- 涉及外部行情源、部署、数据库、认证等未决事项时，必须单独列为“待决策项”。
