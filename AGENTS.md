# AI Industry Chain Atlas Project Rules

## 产品定位

本项目面向产业链研究者和投资者，不面向数据结构开发者。界面优先回答：

- 产业链怎么流动？
- 每个环节有哪些关键细分节点？
- 每个节点有哪些可跟踪公司，A股公司在哪里？
- 哪些节点值得继续查看行情、估值、供需关系和证据？

## 信息展示原则

- 默认展示研究/投资信息，不把内部数据结构作为主信息。避免在主界面显著展示“几级节点”“主链模块数”“细分节点数”“映射数”“已覆盖数”等工程统计。
- 必须保留的结构信息只能小字辅助展示，不能抢占主要视觉空间。
- 中间列表要提高信息密度：节点卡片应直接展示代表公司预览；如果写公司数量，就必须让预览与数量逻辑一致。
- 公司预览规则：公司数不超过 4 家时全部展示；超过 4 家时展示前 4 家并以 `+N` 表示剩余公司。
- 右侧详情用于完整展示当前节点的研究信息和代表公司，不重复解释内部层级结构。
- 行情、估值、供需关系、证据等级等复杂信息放到对应子页面，不在节点库里用长说明干扰扫读。

## UI 文案原则

- 文案短，结论先行。
- 默认使用产业链语言，例如“材料 / 光互联 / 高速激光器 / 代表公司”，少用“stageId / nodeId / 映射 / 二级分类”等实现语言。
- 页面标题、卡片、按钮要让投资者一眼知道下一步能获得什么信息。

## 验证要求

涉及 UI 行为变化时：

- 先补或调整组件测试，确认旧行为失败。
- 再实现最小改动。
- 跑 `npm run typecheck`、`npm run test`、`npm run build`。
- 对可见页面变化，尽量用浏览器实际验证关键交互。
