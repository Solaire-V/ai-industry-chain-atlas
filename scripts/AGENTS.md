# Scripts Rules

## 目录定位

`scripts/` 用于保存项目维护脚本，例如静态数据导出、部署前检查和一次性本地验证工具。

## 使用规则

- 脚本必须可从 `package.json` 调用，避免只靠记忆运行裸命令。
- 脚本不得读取或打印密钥、token、`.env` 内容。
- 涉及真实外部写入、生产数据库或公开部署的脚本，只能做显式 dry-run，真实执行必须由用户单独确认。
- 生成文件应优先写入项目内已有约定目录；如需新增输出目录，先补充对应目录约定。

## 命名约定

- 使用英文小写和连字符，例如 `export-supabase-seed.ts`。
- 面向维护者的函数名使用英文，输出给用户的日志可以使用中文或简洁英文。
